## 状態クラス

[State](@ref bosim::State)クラスとして実装されている。状態オブジェクトは[状態の表現](@ref state_rep)セクションで示した各ピークインデックス\f$m\f$に対応する組\f$\left\{ c_m, \boldsymbol{\mu}_m, \boldsymbol{\Sigma}_m \right\}_{m \in \mathcal{M}}\f$の列として状態を保持する。

## 操作クラス

測定を含む各種操作クラスは[Operation](@ref bosim::operation::Operation)クラスの派生クラスとして名前空間[bosim::operation](@ref bosim::operation)内で実装されている。状態またはピークに対して変更を行う関数オブジェクトであり、状態を引数としてその中の各ピークの平均ベクトルと共分散行列に変更を加えるか、もしくはピークを引数としてその平均ベクトルと共分散行列に変更を加える。メンバとして対象モード数\f$n\f$に対し\f$2n\f$次元操作行列を保持する。

測定の場合は状態オブジェクトで管理されている乱数を用いてサンプリングを行い、測定値を浮動小数点型で返す。測定以外の操作ではvoid関数オブジェクトとして機能する。

## フィードフォワードクラス

[FeedForward](@ref bosim::FeedForward)クラスとして実装されている。回路クラスのとあるパラメータへのFFの情報を保持する。具体的には以下の情報を持つ。

-   回路クラスの持つどの操作（一般に複数）の測定値をFFに使うか
-   回路クラスの持つどの操作（1つ）のどのパラメータにFFを行うか
-   測定値を引数としてFF値を出力とする関数

回路実行時に、これらの情報に基づいてフィードフォワードの値が計算され対象パラメータへ設定される（加算ではない）。

## 回路クラス

[Circuit](@ref bosim::Circuit)クラスとして実装されている。回路オブジェクトは以下の情報を持つ。

-   操作オブジェクトのユニークポインタの列
-   FeedForwardオブジェクトの列

## 結果クラス

[Result](@ref bosim::Result)クラスとして実装されている。結果オブジェクトはショットの結果の列と実行時間の情報を持つ。ショットの結果は回路実行後の以下のオブジェクトを保持する。

-   状態オブジェクト（オプショナル）
-   操作インデックスと測定結果のマップ

## シミュレータクラス

[Simulator](@ref bosim::Simulator)クラスとして実装されている。状態オブジェクトと回路オブジェクトを引数にとって回路を実行するメソッドを持つ。また、結果オブジェクトも持つ。

## シミュレート関数

[Simulate](@ref bosim::Simulate)関数がシミュレーションのエントリーポイントとして実装されており、以下の引数を持つ。

-   シミュレーションの設定。[SimulateSettings](@ref bosim::SimulateSettings)オブジェクトであり、以下のメンバを持つ構造体である。
    -   ショット数
    -   バックエンド。CPUとGPUのどちらを用いるか、どのレベルで並列化を行うかを列挙型[Backend](@ref bosim::Backend)で指定する。
    -   回路実行後の状態を保存する方法。列挙型[StateSaveMethod](@ref bosim::StateSaveMethod)で以下の3通りを選択できる。
        -   状態を保存しない
        -   最初のショットの状態のみ保存する
        -   すべてのショットの状態を保存する
-   回路オブジェクト
-   状態オブジェクト
